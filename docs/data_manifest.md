# Data manifest

The cached embeddings are **not** bundled with the repository (about 17 GB).
Place them in the directory named by `SHARD_DATA`; the fallback search checks
`data/corpus_cache/` and then the legacy `notebooks/_corpus_cache/`. Curated
machine-readable experiment outputs are bundled under `results/`.

## Core embedding cache

The million-document Russian-Wikipedia cache contains L2-normalised arrays:

- `E_docs_e5small_1000000.npy` -- `intfloat/multilingual-e5-small`, `(1000000, 384)`;
- `E_docs_e5base_1000000.npy` -- `intfloat/multilingual-e5-base`, `(1000000, 768)`;
- `E_docs_mpnet_1000000.npy` -- multilingual MPNet, `(1000000, 768)`;
- `E_docs_e5large_1000000.npy` -- `intfloat/multilingual-e5-large`, `(1000000, 1024)`;
- `E_docs_bgem3_1000000.npy` -- `BAAI/bge-m3`, `(1000000, 1024)`.

Files named `E_queries_<enc>_self_q500.npy` contain 500 deterministic
self-retrieval query embeddings. The associated document IDs are generated by
`numpy.random.default_rng(42).choice(N, 500)`. The optional
`corpus_wiki_ru_1000000.pkl` is used only by reference-corpus lookup studies.
Legacy SVD, HNSW and PQ objects under `index_cache/` are derived and can be
rebuilt.

## Current manuscript evidence

Every current output directory contains its own frozen configuration, hashes,
seeds and run log. The important input/output map is:

- `exp23_corrected_score.py` -> `results/exp23_corrected_score/`; cached
  BEIR/MIRACL embeddings and qrels;
- `exp24_partial_alignment.py` ->
  `results/exp24_partial_alignment_main_v2/` and
  `results/exp24_partial_alignment_lowm/`; e5-small document cache;
- `exp25_cross_release_linkage.py` ->
  `results/exp25_cross_release_linkage/`; e5-small document cache;
- `exp26_ckks_blocksimd.py` -> `results/exp26_ckks_blocksimd/`; cached real
  SciFact e5-small/e5-base document and query embeddings;
- `exp27_formal_dp_baseline.py` -> `results/exp27_formal_dp_baseline/`; the
  BEIR/MIRACL cache used by Experiment 23;
- `exp28_cross_release_churn.py` -> `results/exp28_cross_release_churn/`;
  e5-small/e5-base document caches;
- `exp29_shard_vec2text.py` -> `results/exp29_shard_vec2text/`; locally
  cached AG News records, controlled synthetic records, and the named official
  GTR/Vec2Text checkpoints recorded in its manifest;
- `make_fig_maximal_program.py` -> `results/maximal_program_figures/`; reads
  only aggregate CSV summaries from Experiments 26--28.

Embedding arrays are deliberately not copied into result folders. Their
absolute source paths, shapes, byte sizes and/or hashes are recorded by the
corresponding configuration files so a local cache can be audited.

## Legacy outputs

Experiments 12--22 retain the older `results/exp*_outputs/` layout. They are
included for provenance, but their full-rank alignment gate, cosine-only
cross-release metric and uncalibrated Gaussian diagnostic have been superseded
by Experiments 23--29. Baseline Experiments 7--11 similarly retain
`results/exp7_outputs/` through `results/exp11_outputs/`.
